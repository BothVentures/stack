Synergy — 2-Week Roadmap

Frame. Two-week window. Directional, not OKR. Audience = the 5 of us. No new agent builds yet — first we close what's open.

Context

Synergy reached V3 — Post-Harness era on 2026-05-16. The platform now describes itself: every pipeline has an NLAH spec in harnesses/, the venture registry lives in config/ventures.json, execution traces land in Supabase. The operator surface (/mine, /db, /wr, /vu, /au, /upc, /id, /POV, /sc, /md-web, /notion-web, /transcribe, /cookie-monster, /print-to-drive, /sec-audit) is live. The Cloudflare Worker (V2.11 + V2.12 mine-sweeper) handles three roles on one deploy: Read.ai transcript mining, WhatsApp group mining (code-ready), and Synergy in Slack (1:1 + #synergy mentions).

What that masked: six threads opened in parallel and none of them are closed.

  1. Cleanup wave is mid-flight. Asana incomplete: 301 → 224 → in progress. BOTH OS section: 75 → 34 → target ~15. La Chamana: 54 → 33 → 12 applied. Risks register pruned to a Top-10 but de-dup of W21 backfill not finished. vault/transcripts/ may still hold mined: false stragglers the sweeper has flagged.
  2. Access is not resolved. BOTH Claude bot (the Worker's outbound sender) is in 6 venture channels, not all of them. Read.ai webhooks confirmed only for Mariano; Gaston/Rene/Daiana/Andres pending since instructions were sent 2026-05-18. Sync-daemon onboarded only for Mariano + Gaston (Cowork sandbox blockers still open for Dai/Rene/Andres).
  3. Economics are opaque. Anthropic spend, Cloudflare, Asana, Notion, Read.ai, Supabase, OpenAI — costs scattered across owners and accounts. No view of $/transcript-mined or $/Synergy-Slack-turn. No projected burn through July.
  4. Personal vs communication tools are conflated. Every person has Cowork + Asana + Notion + Slack + WhatsApp + Gmail + Calendar + Drive. Nothing says which surface is canonical for what signal. Two Slack bots in the workspace caused a trial-gate failure on 2026-05-24.
  5. The team-facing surface is ambiguous. Notion is partially live, but it is not the team's first stop. Gaston does not start his Monday on Notion. There is no dashboard.
  6. New agent builds are queued behind everything else. /cso V1 code committed but not activated. /ai, /cw, /bw scoped but not started. bv-os-v2 design done, Q-03 target.

The premise of this roadmap: close Phase A (open threads) before opening Phase B (new builds). Two weeks is the right window. After 2026-06-09 we reassess from a clean baseline.

This document is organized into six etapas. They are not sequential weeks — they overlap. Each etapa is a layer of work that needs closure by ~2026-06-09. The order below reflects priority, not chronology.

Risks to this roadmap

  1. A real meeting fails and re-opens the auto-miner trial gate. Etapa 0 (snapshot) calls this out; Etapa 1 cleanup will compete with re-debug time. Mitigation: keep the trial gate strict — partial-is-failed (memory/feedback_partial_is_failed.md) — and don't claim victory early (memory/feedback_premature_victory_declarations.md).
  2. Sync-daemon redesign drags. Daiana onboarding depends on this. If /upc redesign slips, fall back to a Terminal-only /upc for Daiana with a documented recovery path.
  3. Recall.ai migration creep. Tempting because it eliminates Etapa 2's Read.ai webhook chase. But it adds a new external dependency mid-stabilization. Stay deferred.
  4. Bus factor 1 on Mariano. Etapa 2 (Daiana onboarding) is the only structural mitigation in this window. Etapa 5 (Notion as canonical surface) reduces the surface area where Mariano is the only knower.
  5. Drift between CLAUDE.md and reality. Already documented as a structural risk (projects/synergy/status.md). Etapa 6 (/ai Asuntos Internos) is the long-term fix, but it lives outside this window. Mitigation: every commit in this window that touches the operator surface includes a corresponding CLAUDE.md edit.

Closing frame

The premise of this roadmap is restraint. The system grew faster than its hygiene. Two weeks of closing-not-opening returns us to a state where the next build cycle can be deliberate rather than reactive. Nothing here is glamorous. All of it is load-bearing.

After 2026-06-09: reassess. If Etapas 1-5 close clean, the post-window cycle opens /ai Asuntos Internos and /cso activation in parallel. If they don't, we extend the window and stay honest about it.
